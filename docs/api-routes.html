Référence API HTTP

Liste exhaustive des endpoints exposés par pokclock-api (Go/Echo). Pour chaque route : méthode, chemin, authentification requise, paramètres, body JSON, codes de réponse et rôle métier. Voir aussi la version MD pour AI.

Dernière maj 2026-05-28 Phases couvertes 0.A → 0.I Source internal/api/router.go

Conventions

  • Format : toutes les requêtes/réponses en JSON, Content-Type: application/json. Erreurs au format {"error": "code_machine_readable", "detail"?: "message humain"}.
  • Timestamps : RFC3339 UTC (2026-05-28T13:42:00Z).
  • UUIDs : format canonique xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.
  • Auth :
    • anonyme aucun token requis
    • JWT player cookie pokclock_player (HttpOnly, Domain=.pokclock.com)
    • JWT staff header Authorization: Bearer <jwt>, license desktop, club-scoped
    • JWT superadmin rôle superadmin requis, cross-clubs
  • Rate-limit : /api/auth/* et /api/players/auth/* limités à 10 req/min par IP (memory store Echo).

1. Système

anonyme
Réponse

Purpose
Health check. Retourne le build SHA du serveur.

GET/.well-known/jwks.json

Auth
anonyme
Réponse
200 { "keys": [...] } (JWKS RS256)
Purpose
Expose la clé publique pour qu'un consommateur tiers vérifie les JWT émis par l'API.

2. /api/auth/* — Authentification desktop (license → JWT staff)

Échange license + hardwareId contre un JWT RS256 24h. Utilisé par l'app WPF au démarrage. Rate-limit : 10 req/min/IP.

POST/api/auth/issue

Auth
anonyme (vérif côté Worker Cloudflare)
Body
{ "licenseKey": string, "hardwareId": string, "workerToken": string }
Réponse
200 { "token": "JWT", "tier": "Solo|Club|Pro", "expiresAt": RFC3339 }
Erreurs
400 invalid_body, 400 missing_fields, 401 license_invalid, 502 license_verification_unavailable
Purpose
Vérifie la licence côté Worker Cloudflare D1, émet un JWT RS256 pour appeler les routes club-scoped (/api/structures/*, /api/club/*).

POST/api/auth/refresh

Auth
anonyme
Réponse
501 { "error": "refresh_not_yet_implemented", "phase": "0.B" }
Purpose
Stub pour Phase 0.B. La rotation de tokens n'est pas implémentée — re-call /api/auth/issue.

3. /api/structures/* — Structures de blinds (club-scoped)

Auth JWT staff. Filtrage automatique sur claims.ClubID — impossible de voir les structures d'un autre club.

Liste les structures de blinds du club du caller.

GET/api/structures/:slug

Réponse
200 { structure }
Erreurs

PATCH/api/structures/:slug

{ "name"?, "kind"?, "settings"? } tous optionnels

Licenses du club

GET/public/clubs/:slug/tournaments

Réponse
200 { tournaments: [...], club: {id, name, slug} }
Erreurs
404 club_not_found
Purpose
Liste les tournois open|closed du club.

GET/public/clubs/:slug/tournaments/:id

Réponse
200 { ...tournament, players: [], prizeStructure?, pointScheme? }
Purpose
Détail tournoi. La liste players est filtrée selon show_players + players_display_mode (hidden/full_name/first_initial_last/pseudo).

POST/public/clubs/:slug/tournaments/:id/register

Body
{ "firstName": string, "lastName": string, "email": string, "phone"?: string }

400 missing_fields|invalid_email, 404 club_not_found|tournament_not_found, 409 registrations_closed|tournament_started|tournament_full|already_registered, 403 members_only

6. /api/players/auth/* — Authentification joueur

Magic link Resend + Google OAuth. Rate-limit : 10 req/min/IP. Token sortant : cookie pokclock_player (HttpOnly, Secure, SameSite=Lax, Domain=.pokclock.com, 30 jours).

POST/api/players/auth/magic-link

Body
{ "email": string }
Réponse
202 { status: "sent" } (TOUJOURS 202, anti-énumération)
Erreurs
400 missing_email|invalid_email
Purpose
Email Resend avec lien <API>/api/players/auth/magic-link/verify?token=<32 bytes base64url>. Token valide 15 min, single-use.

GET/api/players/auth/magic-link/verify

Query
?token=<base64url>
Réponse
302 Redirect → <publicSiteURL>/joueurs/dashboard + Set-Cookie pokclock_player
Erreurs
Redirect avec ?error=invalid_token|expired_token|used_token
Purpose
Consomme le token, upsert le player, déclenche l'auto-link Phase 0.H, émet JWT.

POST/api/players/auth/logout

Auth
JWT player
Réponse
200 { status: "logged_out" } + clear cookie
Purpose
Logout côté client. Pas de blacklist serveur, JWT expire naturellement.

GET/api/players/auth/google/start

Query
?redirect_uri=<URL>
Réponse
302 Redirect vers consent Google
Erreurs
503 google_oauth_disabled

GET/api/players/auth/google/callback

Query
?code=&state=
Réponse
302 Redirect + Set-Cookie pokclock_player
Purpose
Échange code → id_token, upsert player, déclenche auto-link 0.H, set cookie.

Dashboard agrégé (Phase 0.I)

GET/api/players/me/dashboard

Réponse
200 { widgets: [ClubWidget...] }
Tri
Par dernière activité (max start_at des tournois du club, fallback requested_at de la membership).
Scope
Uniquement memberships status='active'.
Erreurs
401 not_authenticated, 410 player_gone
Purpose
Agrégation pour le dashboard joueur. Un widget par club actif : rang global, saison active + rang scopé, 3 prochains tournois + statut d'inscription.

Structure d'un ClubWidget :

{
  "club": { "id": uuid, "slug": string, "name": string },
  "myRank": {
    "position": int,
    "totalEntries": int,
    "totalPoints": int,
    "tournamentsPlayed": int
  } | null,
  "activeSeason": {
    "id": uuid, "slug": string, "name": string,
    "startsAt": RFC3339, "endsAt": RFC3339,
    "myRank": { ... } | null
  } | null,
  "upcomingTournaments": [{
    "id": uuid, "name": string, "startAt": RFC3339,
    "audience": "public|members_only",
    "spotsLeft": int | null,
    "myRegistrationStatus": "pending|confirmed|rejected|cancelled" | null
  }]
}

Calcul du rang : agrégation SUM(points) groupée par (lower(firstName), lower(lastName)), identique au leaderboard public. Le joueur est matché par player_id OU par nom (insensible à la casse) pour englober les résultats publiés avant la création du compte cloud.

Récap rapide par stack consommateur

App WPF desktop (StackPilot/PokClock)

  • POST /api/auth/issue au démarrage (license → JWT)
  • GET|POST|PATCH|DELETE /api/structures/* (CRUD blinds)
  • POST /api/club/tournaments (publish UPSERT depuis l'app)
  • POST /api/club/tournaments/:id/results (publication des résultats)
  • POST /api/club/tournaments/:id/close
  • GET /api/club/audit, GET /api/club/members, etc.

Site Next.js (pokclock-react)

  • SSR anonyme : GET /public/clubs/:slug/*
  • Auth joueur : POST /api/players/auth/magic-link, GET /verify, GET /api/players/auth/google/start|callback
  • Espace joueur : GET|PATCH|DELETE /api/players/me, GET /api/players/me/{registrations,clubs,dashboard,google-calendar}, POST /api/players/me/clubs/:slug/{join,tournaments/:id/register}, etc.

Worker Cloudflare (license verification)

  • N'appelle pas l'API. C'est l'inverse : POST /api/auth/issue appelle le Worker pour vérifier (licenseKey, hardwareId, workerToken) contre D1.

    • 004_licenses.sql : table licenses (FK clubs CASCADE, license_key UK, worker_token_hash SHA-256, hardware_id nullable, role member/admin/superadmin, status active/revoked/expired, expires_at, last_seen_at, revoked_at)

  • Enrichir les claims JWT
    • auth.Role type avec constantes RoleMember, RoleAdmin, RoleSuperadmin
    • Claims.Role + Claims.ClubID (rétro-compat : tokens existants restent valides)
    • Signer.Issue(IssueParams{}) refactoré (struct au lieu de 4 params positionnels)
    • VerifyResponse (Worker) expose role + clubId (optionnels)
    • Fallback : Worker n'expose pas encore → JWT émis en RoleMember
  • Middleware RBAC dans internal/api/middleware.go
    • requireRole(role) — exige exactement ce rôle
    • requireAnyRole(...) — OU logique
  • Routes API admin (internal/api/admin_clubs.go, admin_licenses.go, admin_audit.go)
    • GET /api/admin/clubs — liste paginée + filtres status/q + pagination
    • POST /api/admin/clubs — création
    • GET /api/admin/clubs/:id — détail
    • PATCH /api/admin/clubs/:id — update name/plan/email/notes
    • POST /api/admin/clubs/:id/suspend + unsuspend
    • DELETE /api/admin/clubs/:id — soft delete
    • GET /api/admin/licenses — liste + filtres clubId/status/role
    • POST /api/admin/licenses — génère license_key POK-XXXX-XXXX-XXXX-XXXX + worker_token 32 bytes hex, stocke uniquement SHA-256 du token
    • POST /api/admin/licenses/:id/revoke
    • GET /api/admin/audit — recherche cross-clubs avec jointure structures, filtres clubId/actor/action/dates/structureId
  • Tests signer_test.go adaptés à la nouvelle signature + nouveau test fallback role=member
